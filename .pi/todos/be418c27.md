## Description

Replace `pi --mode rpc` child process spawning in `rpc-child.ts` with direct `createAgentSession()` from `@mariozechner/pi-coding-agent` SDK.

## Decisions

- **Extensions in children**: Yes — child sessions load global extensions (zotero, brave-search, etc.)
- **Extension UI forwarding**: No — drop the Discord-based UI proxy. Non-interactive sessions return SDK defaults (`false`/`undefined` for confirm/select/input). Removes ~60 lines of complexity.
- **Model**: Children use the SDK default model (based on available API keys), not the master's current model.
- **Session storage**: `SessionManager.inMemory()` for now. File-backed persistence deferred to TODO-050b59b6.
- **Refactor scope**: Large refactor OK — `RpcChild` class internals and `index.ts` integration can both change significantly.

## What Changes

### Remove
- JSONL stdin/stdout protocol parsing and buffer management
- ANSI escape stripping (`ANSI_RE` regex before JSON.parse)
- `resolvePiBinary()` / subprocess spawning
- Extension UI sub-protocol handling (`handleUIRequest`, `handleUIResponse`, `pendingUI` state)
- `PI_RELAY_CHILD` env var guard (replaced by different mechanism)

### Replace With
- `createAgentSession({ sessionManager: SessionManager.inMemory(), ... })` per child
- `session.subscribe()` for typed `AgentSessionEvent` events
- `session.prompt()` / `session.followUp()` for sending messages
- `session.abort()` for cleanup
- `session.dispose()` for teardown

### Keep
- `StreamCoalescer` — still needed for batching Discord message edits
- Discord thread creation and routing logic in `index.ts`
- Tool call summary formatting in `formatter.ts`
- Image upload from tool results

## Key Design Issue: Recursive Extension Loading

Current approach: `PI_RELAY_CHILD=1` env var checked at top of `index.ts`.

With SDK, child sessions run in the same Node.js process — can't use env vars per-session. Options:
1. Custom `ResourceLoader` that filters out pi-relay from discovered extensions
2. Pass extensions list explicitly to `createAgentSession()` excluding pi-relay
3. Module-level flag (e.g. `let isChildContext = false`) checked in the factory function

Need to investigate which the SDK supports cleanly.

## Event Mapping (RPC → SDK)

| Current RPC event | SDK `AgentSessionEvent` | Discord action |
|---|---|---|
| `agent_start` | `agent_start` | Send typing indicator, create coalescer |
| `agent_end` | `agent_end` | Set streaming = false |
| `message_update` (text_delta) | `message_update` (.assistantMessageEvent.type === "text_delta") | Feed delta to StreamCoalescer |
| `turn_end` | `turn_end` | Flush coalescer, upload images, send tool summaries |
| `tool_execution_start` | `tool_execution_start` | Send typing indicator |
| `extension_error` | (extension errors logged by SDK) | Optional: send warning to thread |

## Files to Change

- `src/rpc-child.ts` — full rewrite: remove subprocess, use `AgentSession`
- `src/index.ts` — update `spawn_session` tool, remove `PI_RELAY_CHILD` guard, adjust child creation
- `package.json` — verify `@mariozechner/pi-coding-agent` is a dependency (already is)

## Sketch

```typescript
// rpc-child.ts (new shape)
import { createAgentSession, SessionManager, AuthStorage, ModelRegistry } from "@mariozechner/pi-coding-agent";

export class SessionChild {
  private session: AgentSession | null = null;
  private coalescer: StreamCoalescer | null = null;
  public onExit: (() => void) | null = null;

  constructor(
    private threadId: string,
    private discord: DiscordClient,
    private cwd: string,
  ) {}

  async start(initialTask?: string): Promise<void> {
    const authStorage = AuthStorage.create();
    const modelRegistry = new ModelRegistry(authStorage);

    const { session } = await createAgentSession({
      sessionManager: SessionManager.inMemory(),
      authStorage,
      modelRegistry,
      cwd: this.cwd,
      // resourceLoader: custom loader excluding pi-relay?
    });

    this.session = session;

    session.subscribe((event) => {
      this.handleEvent(event);
    });

    if (initialTask) {
      await session.prompt(initialTask);
    }
  }

  sendPrompt(username: string, content: string, images?: DiscordImage[]): void {
    const message = `[Discord @${username}]: ${content}`;
    if (this.session?.isStreaming) {
      this.session.followUp(message);
    } else {
      this.session?.prompt(message);
    }
  }

  kill(): void {
    this.session?.abort();
    this.session?.dispose();
    this.coalescer?.reset();
    this.onExit?.();
  }

  private handleEvent(event: AgentSessionEvent): void {
    switch (event.type) {
      case "agent_start": ...
      case "message_update": ...
      case "turn_end": ...
      case "tool_execution_start": ...
    }
  }
}
```
