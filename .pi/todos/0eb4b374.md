## Description

Stream `tool_execution_update` events to Discord threads to show tool progress (e.g., partial bash output, file read progress).

## Current State

- `rpc-child.ts` only uses `tool_execution_start` to send a typing indicator
- `tool_execution_update` events with `partialResult` are ignored
- Users see no feedback between tool start and turn end

## Target State

- Stream partial tool output to Discord via the StreamCoalescer
- Especially useful for long-running bash commands where output trickles in
- Could show partial bash output as it arrives

## Implementation Notes

- `tool_execution_update` events contain `partialResult.content` with accumulated output
- Could feed text deltas into the StreamCoalescer for the thread
- Need to be careful not to spam Discord — only update periodically
- Consider only streaming for `bash` tool (most useful), skip for read/edit/write

## Depends On

- Ideally done after TODO-be418c27 (SDK migration) for typed events
- But can also be added to current `processEvent()` in `rpc-child.ts`
