## Architecture

Master pi (interactive mode) manages spawned sessions as RPC child processes. Master owns all Discord I/O.

```
Master pi (interactive, runs relay extension)
‚îú‚îÄ‚îÄ Discord client (single connection)
‚îÇ   ‚îú‚îÄ‚îÄ #channel ‚Üí master pi directly
‚îÇ   ‚îú‚îÄ‚îÄ thread-1 ‚Üí RPC child A (stdin/stdout)
‚îÇ   ‚îú‚îÄ‚îÄ thread-2 ‚Üí RPC child B (stdin/stdout)
‚îÇ   ‚îî‚îÄ‚îÄ thread-3 ‚Üí RPC child C (stdin/stdout)
‚îÇ
‚îú‚îÄ‚îÄ RPC Child A: pi --mode rpc --cwd /project-a
‚îú‚îÄ‚îÄ RPC Child B: pi --mode rpc --cwd /project-b
‚îî‚îÄ‚îÄ RPC Child C: pi --mode rpc --cwd /project-c
```

## Decisions

- Master stays interactive ‚Äî only spawned sessions use RPC mode
- Master's own confirmations stay in TUI (accepted)
- Children die on master restart ‚Äî no persistence (accepted)
- No timeout for Extension UI prompts ‚Äî wait indefinitely for Discord user response
- Thread archive/delete kills child process

## New module: `src/rpc-child.ts`

RpcChild class wrapping a `pi --mode rpc` child process:
- Spawn `pi --mode rpc` with correct cwd
- JSONL reader on stdout ‚Äî parse events
- Route `message_update` ‚Üí coalesce + send to Discord thread
- Route `tool_execution_start/end` ‚Üí tool summaries to Discord
- Route `extension_ui_request` ‚Üí Discord prompt ‚Üí `extension_ui_response` back to stdin
- Send Discord messages ‚Üí `prompt` command on stdin
- Handle `agent_start`/`agent_end` for typing indicators
- Kill child on thread archive/delete

## What gets removed

- Master lock (`relay-master.lock`, `acquireMasterLock`, `releaseMasterLock`)
- Session mode (`isMaster`, `PI_RELAY_THREAD_ID`, `PI_RELAY_INITIAL_TASK_FILE`)
- `relay-mappings.json` persistence
- `session_switch` handler for creating threads
- tmux dependency for spawned sessions

## Implementation steps

1. Create `src/rpc-child.ts` ‚Äî RpcChild class
2. Refactor `spawn_session` ‚Äî use RpcChild instead of tmux
3. Route thread messages ‚Äî discord `onMessage` checks thread‚ÜíRpcChild map, forwards via `prompt`
4. Handle RPC events ‚Äî stream text, tool calls, images to Discord thread
5. Handle Extension UI ‚Äî forward confirm/select/input to Discord, pipe response back (no timeout)
6. Cleanup ‚Äî remove master lock, session mode, mapping persistence
7. Thread lifecycle ‚Äî archive/delete kills child process
