## Description

Handle `auto_compaction_start/end` and `auto_retry_start/end` events from child sessions and surface them as Discord messages in the thread.

## Current State

- `rpc-child.ts` only handles: `agent_start`, `agent_end`, `message_update`, `turn_end`, `tool_execution_start`, `extension_ui_request`, `extension_error`
- Auto-compaction and retry events are silently ignored
- Users have no visibility into why a session might be slow or paused

## Target State

- On `auto_compaction_start`: send "🔄 Compacting context..." to thread
- On `auto_compaction_end`: send "✅ Context compacted (150K → 30K tokens)" or "❌ Compaction failed"
- On `auto_retry_start`: send "⏳ Retrying (attempt 2/3, waiting 2s)..."
- On `auto_retry_end`: send "✅ Retry succeeded" or "❌ Retry failed after 3 attempts"

## Implementation

With SDK migration (TODO-be418c27), these are typed events from `session.subscribe()`:
```typescript
case "auto_compaction_start":
  discord.sendMessage(threadId, "🔄 Compacting context...");
  break;
case "auto_compaction_end":
  if (event.result) {
    discord.sendMessage(threadId, `✅ Context compacted (${event.result.tokensBefore} tokens → compacted)`);
  } else if (event.aborted) {
    discord.sendMessage(threadId, "⚠️ Compaction aborted");
  }
  break;
```

Without SDK migration, add cases to `processEvent()` in `rpc-child.ts`.
