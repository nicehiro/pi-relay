## Decisions

- **Discord → pi images**: fetch Discord attachment URLs, pass to pi as base64 images — needed now
- **Pi → Discord images**: tool results with image content uploaded as Discord file attachments — nice-to-have, design for it but lower priority
- **Large output as files**: not needed — keep current truncation/spoiler approach
- **Embed formatting**: dropped, no use case

## Implementation

- [x] Discord → pi: detect image attachments in incoming messages, fetch and convert to base64, pass as image content in `sendUserMessage()`
- [x] Pi → Discord: detect image content in tool results, upload as Discord file attachments via `AttachmentBuilder`
- [x] Refactored message handler into `createMessageHandler` to avoid duplication between session_start and reconnect
- [x] Added `sendFile` to `DiscordClient` for file uploads
- [x] Messages with only images (no text) handled with `[image]` placeholder
