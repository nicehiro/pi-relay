## Decisions

- **Auth**: user ID allowlist only, silent ignore for unauthorized (no reply)
- **No role-based auth**: user ID sufficient
- **Multi-machine**: each machine has independent `~/.pi/agent/relay.yaml` with its own channel — already works
- **Multi-channel per machine**: dropped — 1 machine = 1 channel, use threads (Phase 5) for session separation
- **Commands**: `/relay reconnect` and `/relay disconnect` already implemented
- **Status widget**: current `setStatus` footer is sufficient
