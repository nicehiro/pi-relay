## Decisions

- **Bot token**: already created, stored in env `DISCORD_BOT_TOKEN` or config YAML
- **Intents**: `Guilds`, `GuildMessages`, `DirectMessages`, `MessageContent` (from pi-messenger-bridge)
- **Ignore bots**: only process human user messages
- **Message queuing**: let pi handle via `sendUserMessage()`, just track which channel ID to reply to
- **Streaming**: use coalesce pattern (flush partial responses during a turn, ~1500 chars or 1s idle)
- **Tool calls**: summary line visible (`🔧 name (arg=val)`), tool output in `||spoiler||` tags
- **Reasoning/internal output**: show everything, folded with spoiler tags by default
- **Reconnection**: rely on discord.js built-in shard reconnect for Phase 1
- **Testing**: local workstation first
