## Scope
Build the core extension: Discord bot connects, receives messages in configured channel, routes to pi, sends responses back.

## Tasks
1. **Project scaffold** — `package.json` (discord.js, yaml, pi deps), `tsconfig.json`, `config.example.yaml`
2. **`src/types.ts`** — `RelayConfig`, `PendingChat` types
3. **`src/config.ts`** — Load YAML config from extension dir, env var fallback for token
4. **`src/formatter.ts`** — `extractText()`, `hasToolCalls()`, `formatToolCalls()`, `splitMessage()` (2000 char limit)
5. **`src/discord.ts`** — `DiscordClient` class: connect with intents (Guilds, GuildMessages, MessageContent), channel filtering, `sendMessage()`, `sendTyping()`, event emitter for incoming messages, dedup, reconnection
6. **`src/index.ts`** — Extension entry point:
   - `session_start`: load config, connect Discord
   - `turn_start`: typing indicator
   - `turn_end`: format + send response, multi-turn tracking (clear pendingChat only when no tool calls)
   - `/relay status` command
   - `session_shutdown`: disconnect

## Acceptance
- Run pi with extension, send message in Discord channel, get response back
- Messages >2000 chars split correctly
- Tool calls shown as summaries (🔧 bash, 🔧 edit, etc.)
- Bot ignores messages in other channels
- Typing indicator while pi processes
